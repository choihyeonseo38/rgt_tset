# 온라인 도서관 관리 시스템 API (Backend)

본 프로젝트는 FastAPI를 활용하여 구축된 도서관 관리 시스템의 백엔드 서버입니다. RESTful API 설계를 바탕으로 사용자 인증, 도서 리소스 관리, 대출 및 반납 비즈니스 로직을 구현하였습니다.

## 주요 구현 내용

### 1. API 엔드포인트 구현 (POST, GET, DELETE)
* **POST**: 회원가입(`/auth/signup`), 로그인(`/auth/login`), 도서 등록(`/books`), 대출 신청(`/loans`) 기능을 구현하였습니다.
* **GET**: 카테고리 및 대출 가능 여부에 따른 도서 검색(`/books`)과 사용자의 대출 현황 조회(`/users/me/loans`) 기능을 포함합니다.
* **DELETE**: 관리자 권한으로 특정 도서 데이터를 삭제하는 기능을 구현하여 리소스 관리 기능을 완성했습니다.

### 2. 데이터베이스 모델
* **메모리 기반 저장소**: 별도의 외부 데이터베이스 설치 없이 Python 딕셔너리 구조(`db`)를 사용하여 데이터를 관리하도록 설계되었습니다.
* **데이터 구조**: 사용자(users), 도서(books), 대출(loans) 정보를 각각의 테이블 구조로 관리하며 고유 ID를 통해 연동됩니다.

### 3. 인증 및 인가 시스템
* **인증(Authentication)**: 로그인 시 아이디와 비밀번호를 확인하고 Bearer 타입의 액세스 토큰을 발급합니다.
* **인가(Authorization)**: 도서 등록 및 삭제, 대출 등 권한이 필요한 작업은 HTTP Header에 유효한 토큰이 포함되어 있는지 확인하는 로직을 적용했습니다.

### 4. 데이터 검증 (Data Validation)
* **Pydantic 활용**: `BaseModel`을 상속받은 클래스를 통해 입력 데이터의 유효성을 검사합니다.
* **무결성 보장**: 잘못된 데이터 형식이나 필수 필드 누락 시 서버에서 자동으로 에러를 반환하여 시스템의 안정성을 확보했습니다.

### 5. 환경 설정 및 실행 방법
* **개발 환경**: Python 3.13+, FastAPI, Uvicorn
* **서버 실행**: `python -m uvicorn main:app --reload` 명령어를 통해 실행합니다.
* **API 문서**: 서버 실행 후 `http://localhost:8000/docs` 주소의 Swagger UI를 통해 모든 API의 명세 확인 및 테스트가 가능합니다.

## 실행 결과 증빙

* **API 명세서**: Swagger UI를 통해 구현된 전체 엔드포인트 목록을 확인할 수 있습니다.
* **서버 로그**: 클라이언트의 API 요청 시 서버 터미널에 `200 OK` 또는 `201 Created` 로그가 정상적으로 출력됨을 확인했습니다.
* **데이터 검증**: 하단의 Schemas 탭에서 정의된 데이터 모델 구조를 확인할 수 있습니다.
